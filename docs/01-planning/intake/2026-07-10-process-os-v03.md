# Goodz Process OS v0.3 기획 입력서

## 배경

Goodz v0.2는 P0–P4 진행률, QA, 릴리스 준비를 대시보드에서 확인할 수 있습니다.  
다음 단계는 실제 스타트업에서 쓰기 좋게 **기획 요청부터 산출물까지 한 흐름으로 정리**하는 것입니다.

참고 방향은 `corp-brain`의 납품 산출물 구조입니다. Goodz는 이를 쇼핑몰 데모가 아니라 스타트업 프로젝트 운영 템플릿으로 흡수합니다.

## 목표

- 기획 입력을 대시보드에서 볼 수 있다.
- PRD, 유저스토리, 화면설계, API, QA, 릴리스 문서를 산출물 레지스트리로 추적한다.
- Phase Gate가 단순 체크박스가 아니라 산출물 완료 상태와 연결된다.
- 에이전트가 다음 작업을 할 때 `status.json`만 읽어도 현재 운영 상태를 이해할 수 있다.

## 범위

### 포함

- `docs/01-planning/intake/` 기획 입력함
- `docs/deliverables/` 산출물 인덱스
- `status.json`의 `intakes`, `deliverables`
- process-dashboard의 `기획`, `산출물` 메뉴

### 제외

- 웹 UI에서 산출물을 직접 편집하는 기능
- 사용자/권한/승인자 DB
- 외부 배포 플랫폼 연동
