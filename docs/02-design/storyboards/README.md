# Storyboards

Goodz 스토리보드는 화면 목록이 아니라, 사용자가 어떤 맥락에서 어떤 결정을 하고 다음 화면으로 넘어가는지 설명합니다.

## SB-001 — 기획 요청에서 디자인 handoff까지

**Actor:** PM / Product Operator

1. 사용자가 새 기능 요청을 `기획` 또는 `변경`에 남깁니다.
2. PM이 `디자인` 메뉴에서 관련 레퍼런스를 확인합니다.
3. PM/Designer가 저해상도 와이어프레임을 먼저 작성합니다.
4. 승인 기준이 필요하면 `승인` 메뉴에 DACI 레코드를 만듭니다.
5. 개발자는 스토리보드와 와이어프레임을 보고 화면 스펙을 작성합니다.
6. 구현 후 `traceLinks`에 commit과 CI 증거를 연결합니다.

**Linked wireframes:** `WF-001`

## SB-002 — 커머스 Admin 화면 개선

**Actor:** Operations Manager

1. 운영자가 상품 목록에서 재고와 가격을 확인합니다.
2. 신규 상품이 필요하면 `+ 상품 등록`으로 이동합니다.
3. 필수 입력과 저장 결과를 명확히 보여줍니다.
4. 오류가 있으면 필드 근처에서 바로 수정합니다.
5. 저장 후 목록에서 새 상품과 재고 상태를 확인합니다.

**Linked wireframes:** `WF-002`
